## 1. Template Overview

Uma página é o componente de nível superior montado pelo TanStack Router para uma determinada rota. Sua única responsabilidade é **layout e composição**. Ela renderiza sub-componentes usando valores e callbacks fornecidos por um hook.

**A regra é simples:** se não é JSX, não pertence ao `{Name}Page.tsx`.

| Arquivo | Responsabilidade |
|---|---|
| `{Name}Page.tsx` | Renderiza JSX, compõe sub-componentes, aplica layout |
| `hooks/use{Name}Page.ts` | Lógica de negócio, estado local, event handlers |
| `hooks/use{Name}Queries.ts` | Hooks TanStack Query escopados à feature |
| `components/` | Sub-componentes privados à feature |

## 2. File Structure

### Página mínima (apenas para páginas extremamente simples, <50 linhas)

```
src/features/{domain}/pages/
└── {Name}Page.tsx
```

### Página padrão (listagem, formulário ou dashboard)

```
src/features/{domain}/
├── pages/
│   └── {Name}Page.tsx            # Máx ~80 linhas — apenas composição
├── components/
│   ├── {Name}Table.tsx           # Colunas da tabela (quando há AppDataTable)
│   └── {Name}Form.tsx            # Formulário com schema (quando há form)
├── data/
│   └── {name}.mock.ts            # Mock data + status configs (temporário)
└── hooks/
    ├── use{Name}Page.ts          # lógica da página
    └── use{Entity}Queries.ts     # queries e mutations
```

### Página complexa com sub-componentes

```
src/features/{domain}/
├── pages/
│   └── {Name}Page.tsx
├── hooks/
│   ├── use{Name}Page.ts
│   └── use{Entity}Queries.ts
└── components/
    ├── {SubFeature}.tsx
    └── {SubFeature}/
        ├── index.tsx
        └── hooks/
            └── use{SubFeature}.ts   # opcional
```

Sub-componentes dentro da feature são **privados** àquela feature. Se um sub-componente for necessário em uma segunda feature, deve ser promovido para `src/components/shared/App{Name}.tsx`.

## 3. Props Typing

Páginas montadas diretamente pelo router geralmente não recebem props externas. Parâmetros de rota são acessados via hooks do TanStack Router.

```tsx
// Parâmetros de rota
const { carId } = carDetailRoute.useParams()

// Search params (query string)
const { page, query } = carsRoute.useSearch()
```

Sub-componentes de página recebem props tipadas explicitamente:

```tsx
interface CarsTableProps {
  data: Vehicle[]
  isLoading: boolean
  onRowClick: (id: string) => void
}
```

## 4. Component Structure

```tsx
// src/features/{domain}/pages/{Name}Page.tsx

import { AppDataTable } from "@/components/shared/AppDataTable"
import { AppButton } from "@/components/shared/AppButton"
import { use{Name}Page } from "../hooks/use{Name}Page"
import { {SubComponent} } from "../components/{SubComponent}"

export function {Name}Page() {
  const {
    data,
    total,
    isLoading,
    page,
    handlePageChange,
    handleSearch,
    handleCreate,
  } = use{Name}Page()

  return (
    <div className="space-y-6">
      {/* Cabeçalho da página */}
      <div className="flex items-center justify-between">
        <div>
          <h1 className="text-2xl font-bold text-foreground">{Title}</h1>
          <p className="text-sm text-muted-foreground">{Subtitle}</p>
        </div>
        <AppButton onClick={handleCreate}>Nova entrada</AppButton>
      </div>

      {/* Conteúdo principal */}
      <AppDataTable
        data={data}
        total={total}
        isLoading={isLoading}
        page={page}
        onPageChange={handlePageChange}
        onSearch={handleSearch}
        columns={columns}
      />
    </div>
  )
}
```

### O que `{Name}Page.tsx` NÃO deve conter

- `useState` (exceto em casos raros onde o estado é puramente cosmético e não pertence ao hook)
- `useEffect` com lógica não-trivial
- Funções de event handler definidas inline — estas vivem no hook
- Lógica de filtragem, paginação ou transformação de dados
- Chamadas diretas à API ou serviços
- Chamadas diretas ao Zustand store (exceto leitura simples com seletor)
- Estilos inline com `style={{}}`

## 5. Separating UI from Logic

### A fronteira

```
{Name}Page.tsx        →  O que é renderizado
hooks/use{Name}.ts    →  Por que é renderizado, o que acontece quando o usuário age
hooks/use{Name}Queries.ts  →  De onde vêm os dados
```

### Os dados fluem em uma direção

```
Serviço (API)  →  TanStack Query  →  Hook de página  →  Página  →  Componentes
   (HTTP)          (cache)           (lógica/estado)    (JSX)      (visual)
```

A página é o último passo. Ela recebe tudo que precisa do hook e passa valores para sub-componentes via props. Sub-componentes não acessam o store nem chamam hooks de query diretamente — recebem todos os dados via props.

### Exceção: componentes de seção com estado próprio

Para páginas como Dashboard com seções independentes, cada seção pode ter seu próprio hook de dados:

```tsx
// DashboardPage.tsx — orquestra seções independentes
export function DashboardPage() {
  return (
    <div className="space-y-6">
      <KpiSection />       {/* usa useKpiMetrics() internamente */}
      <RecentSalesSection /> {/* usa useRecentSales() internamente */}
      <TaskListSection />  {/* usa usePendingTasks() internamente */}
    </div>
  )
}
```

## 6. Layout Guidelines

- O elemento raiz da página é sempre uma `<div>` com `className="space-y-6"` ou similar
- Cabeçalho da página (título + ação primária) no topo, separado do conteúdo
- Usar `Card` do shadcn/ui para agrupar seções de conteúdo
- Estados de carregamento renderizam skeletons via `isLoading` nos componentes filhos — não spinners globais
- Layout em grid para dashboards: `grid grid-cols-1 gap-6 lg:grid-cols-3`
- Sem estilos inline — todas as classes via Tailwind

```tsx
// Layout típico de página de listagem
<div className="space-y-6">
  {/* Header */}
  <div className="flex items-center justify-between">
    <div>
      <h1 className="text-2xl font-bold text-foreground">Veículos</h1>
      <p className="text-sm text-muted-foreground">Gerencie o estoque de veículos</p>
    </div>
    <AppButton>Novo Veículo</AppButton>
  </div>

  {/* Tabela */}
  <AppDataTable {...tableProps} />
</div>

// Layout típico de dashboard
<div className="space-y-6">
  {/* KPIs */}
  <div className="grid grid-cols-1 gap-4 sm:grid-cols-2 xl:grid-cols-4">
    <StatCard ... />
  </div>

  {/* Conteúdo em duas colunas */}
  <div className="grid grid-cols-1 gap-6 lg:grid-cols-3">
    <div className="lg:col-span-2">...</div>
    <div>...</div>
  </div>
</div>
```

## 7. Complete Example

Uma página de configurações de veículo que compõe um formulário e um modal de confirmação de exclusão.

**`src/features/cars/hooks/useCarDetailPage.ts`**

```tsx
import { useState, useCallback } from "react"
import { useNavigate } from "@tanstack/react-router"
import { toast } from "sonner"

import { useCar } from "./useCars"
import { useUpdateCar, useDeleteCar } from "./useCarMutations"
import { carDetailRoute } from "@/router"
import type { UpdateCarDTO } from "../services/cars.service"

export function useCarDetailPage() {
  const { carId } = carDetailRoute.useParams()
  const navigate = useNavigate()

  const { data: car, isLoading } = useCar(carId)
  const { mutate: updateCar, isPending: isUpdating } = useUpdateCar()
  const { mutate: deleteCar, isPending: isDeleting } = useDeleteCar()

  const [isDeleteModalOpen, setIsDeleteModalOpen] = useState(false)

  const handleUpdate = useCallback(
    (dto: UpdateCarDTO) => {
      updateCar(
        { id: carId, data: dto },
        {
          onSuccess: () => toast.success("Veículo atualizado!"),
          onError: () => toast.error("Erro ao atualizar"),
        },
      )
    },
    [carId, updateCar],
  )

  const handleDelete = useCallback(() => {
    deleteCar(carId, {
      onSuccess: () => {
        toast.success("Veículo removido!")
        navigate({ to: "/cars" })
      },
    })
  }, [carId, deleteCar, navigate])

  return {
    car,
    isLoading,
    isUpdating,
    isDeleting,
    isDeleteModalOpen,
    handleUpdate,
    handleDelete,
    openDeleteModal: useCallback(() => setIsDeleteModalOpen(true), []),
    closeDeleteModal: useCallback(() => setIsDeleteModalOpen(false), []),
  }
}
```

**`src/features/cars/pages/CarDetailPage.tsx`**

```tsx
import { AppButton } from "@/components/shared/AppButton"
import { useCarDetailPage } from "../hooks/useCarDetailPage"
import { CarForm } from "../components/CarForm"
import { DeleteCarModal } from "../components/DeleteCarModal"

export function CarDetailPage() {
  const {
    car,
    isLoading,
    isUpdating,
    isDeleting,
    isDeleteModalOpen,
    handleUpdate,
    handleDelete,
    openDeleteModal,
    closeDeleteModal,
  } = useCarDetailPage()

  if (isLoading) {
    return <div className="animate-pulse space-y-4">{/* skeleton */}</div>
  }

  return (
    <div className="space-y-6">
      <div className="flex items-center justify-between">
        <div>
          <h1 className="text-2xl font-bold text-foreground">{car?.name}</h1>
          <p className="text-sm text-muted-foreground">{car?.plate}</p>
        </div>
        <AppButton variant="destructive" onClick={openDeleteModal}>
          Excluir
        </AppButton>
      </div>

      <CarForm
        defaultValues={car}
        isSubmitting={isUpdating}
        onSubmit={handleUpdate}
      />

      <DeleteCarModal
        isOpen={isDeleteModalOpen}
        isDeleting={isDeleting}
        onConfirm={handleDelete}
        onClose={closeDeleteModal}
      />
    </div>
  )
}
```

## 8. Checklist

Antes de submeter uma nova página, verifique:

- [ ] Arquivo em `src/features/{domain}/pages/{Name}Page.tsx`
- [ ] Named export — sem `export default`
- [ ] `{Name}Page.tsx` contém apenas JSX e desestruturação de props — sem lógica
- [ ] Lógica de negócio está no hook (`hooks/use{Name}Page.ts`)
- [ ] Hooks TanStack Query em arquivo próprio quando a página tem muitas queries
- [ ] Parâmetros de rota acessados via `{route}.useParams()` — não via props
- [ ] Navegação feita via callbacks retornados pelo hook
- [ ] Estado de carregamento delegado aos componentes filhos via `isLoading`
- [ ] Layout segue as grid guidelines (space-y-6, grid para dashboard)
- [ ] Sem estilos inline
- [ ] Rota registrada em `src/router/index.tsx`
- [ ] Sem `console.log` — apenas `console.warn` / `console.error`
- [ ] Sem tipos `any`
- [ ] Testada em light mode e dark mode
