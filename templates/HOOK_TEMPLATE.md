## 1. Template Overview

```tsx
import { useState, useCallback, useMemo } from "react"
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query"

// Tipos definidos inline para hooks simples, ou importados de arquivo separado para hooks complexos

export function use{HookName}(param: ParamType): ReturnType {
  // 1. Store externa (Zustand selectors)
  // 2. Queries e mutations (TanStack Query)
  // 3. Estado local (useState, useReducer)
  // 4. Valores derivados (useMemo)
  // 5. Callbacks (useCallback)
  // 6. Effects (useEffect) — ao final

  return {
    // apenas valores e callbacks nomeados
  }
}
```

O valor de retorno é sempre um **objeto nomeado**. Hooks nunca retornam JSX, instâncias de classe ou tuplas posicionais (exceto para hooks simples de toggle que espelham a API do `useState`).

---

## 2. Naming Convention

Todos os hooks começam com `use`. O restante do nome descreve o que o hook encapsula.

| Propósito | Padrão | Exemplo |
|---|---|---|
| Lógica de página completa | `use{PageName}` | `useCarsPage` |
| Lógica de formulário | `use{Feature}Form` | `useCreateCarForm` |
| Query de detalhe | `use{Entity}` | `useCar` |
| Query de listagem | `use{Entities}` | `useCars` |
| Mutation de criação | `useCreate{Entity}` | `useCreateCar` |
| Mutation de edição | `useUpdate{Entity}` | `useUpdateCar` |
| Mutation de exclusão | `useDelete{Entity}` | `useDeleteCar` |
| Ação específica | `use{Action}{Entity}` | `useApproveSaleOrder` |
| Comportamento reutilizável | `use{Behavior}` | `useDebounce`, `useDisclosure` |

## 3. File Placement

| Escopo | Local |
|---|---|
| Usado em uma única página | `src/features/{domain}/hooks/use{Name}.ts` |
| Queries específicas de domínio | `src/features/{domain}/hooks/use{Entity}Queries.ts` |
| Usado em dois ou mais domínios | `src/hooks/use{Name}.ts` |
| Comportamento genérico de UI | `src/hooks/use{Name}.ts` |

Um hook que começa em um domínio e passa a ser necessário em um segundo deve ser movido para `src/hooks/` — não duplicado.

## 4. Internal State

### `useState` — para valores escalares independentes

```tsx
const [isModalOpen, setIsModalOpen] = useState(false)
const [selectedId, setSelectedId] = useState<string | undefined>(undefined)
```

### `useReducer` — quando múltiplos valores mudam juntos

Use `useReducer` quando dois ou mais campos de estado mudam em resposta ao mesmo evento. Isso evita atualizações de estado parciais e torna as transições explícitas.

```tsx
type State = {
  isSubmitting: boolean
  hasError: boolean
  step: number
}

type Action =
  | { type: "SUBMIT_START" }
  | { type: "SUBMIT_SUCCESS" }
  | { type: "SUBMIT_ERROR" }
  | { type: "NEXT_STEP" }

const reducer = (state: State, action: Action): State => {
  switch (action.type) {
    case "SUBMIT_START":
      return { ...state, isSubmitting: true, hasError: false }
    case "SUBMIT_SUCCESS":
      return { ...state, isSubmitting: false }
    case "SUBMIT_ERROR":
      return { ...state, isSubmitting: false, hasError: true }
    case "NEXT_STEP":
      return { ...state, step: state.step + 1 }
    default:
      return state
  }
}
```

### `useMemo` — para valores derivados custosos

Use `useMemo` apenas quando o cálculo for genuinamente custoso ou quando o valor derivado for passado como prop para um componente filho memoizado.

```tsx
const availableCars = useMemo(
  () => cars?.filter((c) => c.status === "available") ?? [],
  [cars],
)
```

## 5. Exposed Functions

Toda função retornada de um hook deve ser envolida em `useCallback`. Isso evita que o componente consumidor re-renderize a cada avaliação do hook.

```tsx
const handleSubmit = useCallback(
  (formData: CreateCarDTO) => {
    createCar(formData)
  },
  [createCar],
)

const handleClose = useCallback(() => {
  setIsModalOpen(false)
}, [])
```

### Dependency arrays

- Liste todo valor do escopo externo que o callback lê ou chama.
- Se uma dependência é uma referência estável (funções do Zustand, constantes), é seguro incluí-la — não causará re-renders.
- Nunca suprima avisos de lint sobre dependências faltantes. Corrija o array de dependências ou refatore o callback.

---

## 6. Return Structure

Hooks sempre retornam um objeto plano com chaves nomeadas. Isso torna a desestruturação no call site legível e permite ao consumidor pegar apenas o que precisa.

```tsx
return {
  // Dados
  cars,
  total,
  isLoading,
  isError,

  // Valores derivados
  availableCount,
  canCreateNew,

  // Callbacks
  handleCreate,
  handleDelete,
  handleSearch,

  // Setters de estado de UI expostos intencionalmente
  openDeleteModal,
  closeDeleteModal,
}
```

### Convenções de nomeação para valores de retorno

| Tipo | Convenção | Exemplo |
|---|---|---|
| Dados | substantivo | `cars`, `customer`, `saleOrders` |
| Flag de carregamento | `is` + estado | `isLoading`, `isSubmitting` |
| Condição booleana | `is` / `has` / `can` | `isAuthenticated`, `hasError`, `canDelete` |
| Event handler | `handle` + ação | `handleSubmit`, `handleDelete` |
| Imperativo abrir/fechar | `open` / `close` + alvo | `openModal`, `closeDeleteModal` |
| Setter exposto intencionalmente | `set` + valor | `setActiveTab` |

## 7. Complete Example

Hook de página para a tela de configurações de veículo. Coordena queries, mutations, estado de UI local e callbacks de navegação.

**`src/features/cars/hooks/useCarSettings.ts`**

```tsx
import { useState, useCallback, useMemo } from "react"
import { useNavigate } from "@tanstack/react-router"
import { useQueryClient } from "@tanstack/react-query"
import { toast } from "sonner"

import { useAuthStore } from "@/store/auth.store"
import { useCar } from "./useCars"
import { useUpdateCar, useDeleteCar } from "./useCarMutations"
import { carsKeys } from "./carsKeys"
import type { UpdateCarDTO } from "../services/cars.service"

export function useCarSettings(carId: string) {
  // ─── Store externa ──────────────────────────────────────────────────────────
  const user = useAuthStore((s) => s.user)

  // ─── Queries ────────────────────────────────────────────────────────────────
  const { data: car, isLoading } = useCar(carId)

  // ─── Mutations ──────────────────────────────────────────────────────────────
  const { mutate: updateCar, isPending: isUpdating } = useUpdateCar()
  const { mutate: deleteCar, isPending: isDeleting } = useDeleteCar()

  // ─── Estado local ────────────────────────────────────────────────────────────
  const [isDeleteModalOpen, setIsDeleteModalOpen] = useState(false)

  // ─── Valores derivados ───────────────────────────────────────────────────────
  const canEdit = useMemo(
    () => user?.employee?.role === "admin" || user?.employee?.role === "manager",
    [user],
  )

  // ─── Callbacks ───────────────────────────────────────────────────────────────
  const navigate = useNavigate()

  const handleUpdate = useCallback(
    (dto: UpdateCarDTO) => {
      updateCar(
        { id: carId, data: dto },
        {
          onSuccess: () => toast.success("Veículo atualizado!"),
          onError: () => toast.error("Erro ao atualizar veículo"),
        },
      )
    },
    [carId, updateCar],
  )

  const handleDelete = useCallback(() => {
    deleteCar(carId, {
      onSuccess: () => {
        toast.success("Veículo removido!")
        navigate({ to: "/cars" })
      },
      onError: () => toast.error("Erro ao remover veículo"),
    })
  }, [carId, deleteCar, navigate])

  const openDeleteModal = useCallback(() => setIsDeleteModalOpen(true), [])
  const closeDeleteModal = useCallback(() => setIsDeleteModalOpen(false), [])

  // ─── Return ──────────────────────────────────────────────────────────────────
  return {
    car,
    isLoading,
    isUpdating,
    isDeleting,
    canEdit,
    isDeleteModalOpen,
    handleUpdate,
    handleDelete,
    openDeleteModal,
    closeDeleteModal,
  }
}
```

## 8. Checklist

Antes de submeter um novo hook, verifique:

- [ ] Nome começa com `use`
- [ ] Implementado como função nomeada com named export
- [ ] Colocado no arquivo correto (`features/{domain}/hooks/` ou `src/hooks/`)
- [ ] Retorna um objeto nomeado — não JSX, não tupla posicional
- [ ] Todas as funções retornadas estão envolidas em `useCallback`
- [ ] Dependency arrays são completos e corretos
- [ ] Valores derivados custosos estão envolidos em `useMemo`
- [ ] Sem chamadas diretas ao Axios — todo acesso à rede via serviços e TanStack Query
- [ ] Sem `console.log` — apenas `console.warn` / `console.error`
- [ ] Sem tipos `any` — parâmetros e retornos explicitamente tipados
- [ ] Feedback de UI (toasts, modais) disparados via estado ou callbacks `onSuccess`/`onError`
- [ ] Navegação feita via `useNavigate` dentro do hook ou passada como callback ao componente
